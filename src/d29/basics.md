# ACL 基础知识

ACL 的作用，是过滤通过咱们路由器的流量。我（作者）尚且不知道，有哪个网络要允许任何流量类型进入或离开他。

除了过滤流量，ACL 还可用于引用 NAT 池、过滤咱们的调试命令，以及路由映射（这不在 CCNA 考试大纲要求范围内）。根据咱们配置的 ACL 类型，咱们可基于源网络或 IP 地址、目的网络或 IP 地址、协议或端口号等过滤流量。咱们可应用 ACL 到任何路由器接口，包括咱们的 Telnet (`vty`) 端口。


ACL 的三种主要类型如下：

- 标准编号的
- 扩展编号的
- 标准或扩展命名的


所谓标准编号的 ACL，是咱们可以应用到路由器的最基本 ACL 形式。虽然他们最容易配置，但他们也仅有着最受限的可用过滤器。他们只能根据源 IP 地址或网络过滤。识别标准 ACL 的方法，是看配置行前的编号；这些编号将是从 1 到 99。

扩展编号的 ACL，允许精细得多的过滤，但要配置和故障排除他们，则可能更棘手。他们可以过滤目的或源 IP 地址或网络、协议类型及端口号。咱们可用于配置扩展 ACL 的编号，为 100 至 199（包含 100 和 199）。

命名 ACL 允许咱们将过滤器列表与名字，而不是编号关联。这点使他们在路由器配置中更易于识别。命名的 ACL 实际上既可以是扩展的，也可以是标准的；由咱们选择，这种 ACL 初始配置行为何。

为了在 CCNA 考试中取得成功，并成为一名新的思科工程师，咱们需要掌握以下内容：

- 端口号
- ACL 规则
- ACL 的命令语法
