# 地址类

咱们需要知道这点，但却不知道！我（作者）知道，却帮不上什么忙，而地址类别实际上有一些历史意义，因此作为一名新晋思科工程师，当咱们看到一些旧规则，并试图将其应用于新的网络设计方法时，咱们可能会感到困惑。

我们仍将 IP 地址组称为类，但随着子网掩码和 VLSM 的引入，他们实际上已不再适用于网络设计。不过，了解地址类还是很有用的，因为他们会告诉我们，我们可以和不可以将 IP 地址的哪些部分，用于咱们的迷你网络（子网）。

IPv4 发明之初，地址即被划分为不同的类。然后按需将这些地址类别，分配给家公司机构。公司机构越大，地址类就越大。这些地址类分配了字母，从 A 到 E。A 类地址为一些最大的网络保留。A 类地址第一个八位组的编号从 1 到 126 不等。这样做的原因，是第一个八位组的第一位必须为 `0`。当咱们在第一个八位上为 `0` 时，那么其余值就只能从 1 到 126 了，比如：

<code>
<u>0</u>0000001 = 1
<u>0</u>1111111 = 126
</code>

咱们不能有网络上全 0 的地址。当咱们真正加上其他三个八位组时，就可以看到完整的 A 类地址了，例如：

```
10.1.1.1
120.2.3.4
127.0.0.1
```

这些都属于 A 类地址，因为他们在 1 到 126 范围内，127 不是个 IP 地址允许的数字； `127.0.0.1` 实际上是个用于测试咱们设备上 TCP/IP 是否工作的地址。


B 类地址的第一个八位组的前两位，必须设置为 `10`。这意味着第一个八位组就只能使用 128 至 191 的数字，例如：

<code>
<u>10</u>000000 = 128
<u>10</u>111111 = 191
</code>

对于 C 类地址，第一个八位组的前三位，必须设置为 `110`，从而给到咱们 192 至 223 的地址，例如：

<code>
<u>110</u>00000 = 192
<u>110</u>11111 = 223
</code>


D 类地址范围为 `224.0.0.0` 至 `239.255.255.255`，并用于多播（定向的广播）。E 类地址范围从 `240.0.0.0` 到 `255.255.255.254`，并仅供试验用途。

## 子网掩码入门

早先我（作者）曾提到，地址的一个部分标识了网络，另一部分标识了网络上的主机。子网掩码就确定了哪个部分为何。难点在于，光看子网掩码并不总是很容易确定，哪一部分为何。这需要练习，而对于一些较难的地址，咱们必须手工计算出来（或使用子网计算器作弊）。

即使咱们没有把咱们的网络分割成一些较小部分，咱们也将子网掩码，应用到每个用到的地址。每个网络类别，都有个默认的子网掩码，例如：

```
A 类 = 255.0.0.0
B 类 =255.255.0.0
C 类 = 255.255.255.0
```

当那些二进制位开启时，网络就知道这个数字是用于网络，而不是网络上的主机，如下所示：


| 192 | 168 | 12 | 2 |
| :-: | :-: | :-: | :-: |
| 255 | 255 | 255 | 0 |
| 网络 | 网络 | 网络 | 主机 |

上面这个地址表示 `192.168.12` 为网络，`2` 是该网络上的某个主机。此外，任何以 `192.168.12` 开头的地址，都在同一个网络上。从第一个八位组中的数字，与这个默认子网掩码，咱们便可看出这是个 C 类网络。


请记住我（作者）早先提到的规则：咱们不能将网络编号用于主机，因此下面的编号就不能在设备上使用：

```
10.0.0.0
192.168.2.0
174.12.0.0
```

另一条规则是，咱们不能使用各个网络或子网上的广播地址。广播地址指向该网络上的所有设备，因此从逻辑上讲，他不能用于设备。广播地址是其中所有主机位出于活动或开启状态的一个地址：

```
10.255.255.255
192.168.1.255
```

在上面的示例中，主机部分的每个二进制位都是打开的。
