# 使用 IP 地址


接下来，是使用 IP 地址的实际问题 -- 哪些可以使用，哪些不能使用呢？

咱们知道，在过去的三十年里，在计算机使用上有了巨大爆发。个人电脑曾经非常昂贵，很少有人买得起；因此，他们只为那些资金雄厚的公司机构保有。如今，几乎每家每户都有一台或多台计算机。


当然，问题在于 IPv4 是在只有有限数量设备使用的时候设计出来的，而且当时并没有预料到这种情况会发生变化。这种情况会发生变化。在地址分配时，人们意识到，按照目前的增长速度，可用地址很快就会用完。


## 私有 IP 地址

几种解决方案之一，便是保留一些只要该地址不在互联网上使用，任何人便都可使用的地址类别。该地址范围，称为私有 IP 地址，
这一解决方案由两个 RFC，[1918](https://datatracker.ietf.org/doc/html/rfc1918) 与 [4193](https://datatracker.ietf.org/doc/html/rfc4193) 创建出来。作为复习，RFC 是指 Request for Comments，是工程师提交网络方法、协议和技术演进等的想法的一种方式。

私有地址的范围如下：

- `10.x.x.x` -- 以 10 开头的任意地址
- `172.16.x.x` 至 `172.31.x.x` -- 以 `172.16` 至 `172.31` 开头的任意地址，包含 `172.16` 和 `172.31`
- `192.168.x.x` -- 以 `192.168` 开头的任意地址
