# 用于 IPv6 的 EIGRP

尽管用于 IPv6 的 EIGRP 并未在新的 CCNA 考试大纲中明确列出，但出于数个原因，其仍将在这一教学模组中得以介绍。

首先，高度关注 EIGRP 与 IPv6 技术的一些 CCNA 主题，以及一些 EIGRPv6 的主题可能出现于考试中，尽管这种可能性不大。

其次，这一主题相对简单明了，因此要掌握他并不需要花很长时间，特别是考虑到讲解不会很深入。

除了那些开放标准协议外，思科专有的 EIGRP 也已被修改为支持 IPv6。这个 EIGRP 的修改版本，因其对 IPv6 的支持，有时被称为 EIGRPv6，而并非因为其是 EIGRP 路由协议的第 6 次修订。同样，用于 IPv4 的 EIGRP，有时也被称为 EIGRPv4，以区分受两个版本支持的路由协议版本。


总体而言，EIGRPv6 保留了与 EIGRPv4 同样的一些基本核心功能。例如，两个版本仍使用 DUAL 确保无环路的路径，且两个协议都使用组播数据包发送更新 —— 尽管 EIGRPv6 使用 IPv6 组播地址 `FF02::A`，而非由 EIGRPv4 使用的 `224.0.0.10` 组地址。尽管那些同样的核心功能得以保留，但这两个版本之间仍有一些差别。下表 22.6 列出了 EIGRPv4 与 EIGRPv6 之间，或更直接更常用地讲，用于 IPv4 的 EIGRP 与用于 IPv6 的 EIGRP 之间的区别：

**表 22.6** -- **EIGRPv4 与 EIGRPv6 的差异**

| 协议特征 | 用于 IPv4 的 EIGRP | 用于 IPv6 的 EIGRP |
| :-- | :-- | :-- |
| 自动的汇总 | 支持 | 不适用 |
| 身份验证或安全性 | MD5 | 内建于 IPv6 |
| 对等点处于同一子网 | 要求 | 不要求 |
| 通告的内容 | 子网/掩码 | 前缀/长度 |
| 数据包的封装 | IPv4 | IPv6 |

**注意**：由于 EIGRPv6 使用邻居的链路本地地址作为下一跳地址，因此为了两台位于同一自治系统内，且在某一共同网段上的两台路由器之间的邻居关系得以建立，两台路由器的全球 IPv6 单播子网无需相同。这是 EIGRPv4 与 EIGRPv6 之间最显著差异之一，前者要求两个邻居必须在共同的某个子网上，而 EIGRPv6 则通过将链路本地地址用于邻居关系，消除了这一需求。
