# 思科 IOS 软件 EIGRPv4 与 EIGRPv6 的配置差异

在思科 10S 软件中，EIGRPv4 与 EIGRPv6 的配置存在一些显著差异。首要区别在于路由协议的启用方式。对于 EIGRPv4，要启用 EIGRPv4 的路由，并指定出 EIGRPv4 的自治系统编号（ASN），那么 `router eigrp [ASN]` 这条全局配置命令是必需的。在配置 EIGRPv6 时，要启用 EIGRPv6 并指定出本地路由器的 ASN，那么 `ipv6 router eigrp [ASN]` 这条全局配置命令就是必需的。

虽然启用 EIGRPv4 与 EIGRPv6 在某种程度上相似，但一旦路由进程已被启用，在协议状态方面存在显明与显著不同。默认情况下，在 EIGRPv4 启用后，这一协议会自动启动，并假设在正确配置下，就会开始在所有指定运行接口上发送 `Hello` 报文。而在 Cisco IOS 软件中启用 EIGRPv6 时，默认情况下，在该协议已被启用后，其会保持于关闭状态。这意味着即使已在指定接口下启用，EIGRP 进程将不在运行中，直到 `no shutdown` 这条路由器配置命令被执行。

EIGRPv4 与 EIGRPv6 还有另一配置差别，便是在 EIGRPv6 下，路由器 ID 属于强制性的，并必须以 IPv4 的点分十进制表示法予以指定。在指派路由器 ID 时，要记住这个地址不必是某个可路由或可达的地址。

**注意**：当本地路由器上配置了带有 IPv4 地址的任何接口时，那么该路由器将从这些接口中选择路由器 ID —— 优先选择那些环回接口，并当路由器上没有配置的或运行的环回接口时，那么就会使用物理接口。当环回接口中最高的 IP 地址处于 `up` 状态，则就将被选取。若没有环回接口处于 `up` 状态，那么 RID 将选取自处于 `up` 状态的物理接口中，最高的 IP 地址。当上述两种接口均未在路由器上配置时，那么就必须使用 `eigrp router-id [IPv4 Address]` 这一命令。
