# IPv4 与 IPv6 的双栈部署

在双栈的部署下，虽然某些主机具备同时使用 IPv4 和 IPv6 两种协议栈的能力，但在确定何时要使用 IPv6 协议栈，而非 IPv4 协议栈方面，他们仍需一些帮助。幸运的是，在使用如下所述的两种方式之一下，这是可行的：

- 第一种方式需要用户的手动配置。当用户知道目的 IPv6 主机的 IPv6 地址时，他们便可使用这个地址，在他们的双栈主机上手动建立到该远端主机的一个 IPv6 会话。尽管这种方式工作很好，但要记住多个主机的 IPv4 和 IPv6 地址，就会变得相当麻烦；
- 第二种方式会借助于诸如 DNS 的某种名字服务。在这一方式下，比如 www.howtonetwork.com 等的一些完全合格域名（FQDN），会同时 IPv4 和 IPv6 的地址予以配置。这种 FQDN 会以一个 IPv4 协议栈的 A 记录，与一个 IPv6 协议栈的 AAAA 记录表示，这就允许 DNS 服务器，可使用 IPv4 或 IPv6 协议得以查询。
